# v1.2.0-rc.1 — cruft-only skill prune

## What changed

Archives 2 cruft skills from `packages/skills/`. Skill count: **107 → 105**.

- **configure-ecc** — installer for an unrelated project (`everything-claude-code`); its Step 0 `git clone`s a different repo into `/tmp`. Never belonged in a donnyclaude distribution. Category: `cruft:unrelated-project-installer`.
- **continuous-learning** — version-superseded duplicate of `continuous-learning-v2` (v2.1.0 is a strict superset: 100% reliable PreToolUse/PostToolUse hook observation, atomic instincts with confidence scoring, project scoping, and six commands). v1's own SKILL.md frames v2 as the more sophisticated successor. Category: `cruft:version-superseded`.

Per-skill rationale, archive paths, and copy-pasteable restore commands live in [`docs/PRUNE-LOG.md`](../PRUNE-LOG.md) — the canonical rationale document for this prune.

## Deferred: training-duplicate prune

The broader training-duplicate prune originally scoped for v1.2 (20-30 language-pattern and methodology skills) is **deferred to v1.3 pending rubric redesign**.

During Phase 1 execution, the 5-skill calibration pre-flight of the training-duplicate rubric surfaced that clause (c) — "no internal references to this skill exist in donnyclaude's distribution" — cannot distinguish training-duplicate skills from catalog cross-links in the current codebase. Every candidate skill has the same bare-pointer referrer pattern (reviewer agents, rule files, and commands all use `see skill: X` catalog entries rather than semantic dependencies). The rubric as written keeps all of them; a rubric that excludes bare pointers would prune all of them including skills the plan explicitly protected (tdd-workflow and e2e-testing in D-14). v1.3 will research what "semantic dependency" means in a catalog-linked distribution and redesign the rubric from there.

The calibration gate worked as designed — adding the 5-skill pre-flight prevented v1.2 from shipping 41 incorrectly-verdicted skills. Partial audit artifacts are preserved at `.planning/research/v1.3-seeds/` as v1.3 research inputs.

## Install

```
npx donnyclaude@rc
```

This RC is published under the `rc` dist-tag (D-19). That means:

- `npx donnyclaude` (no `@rc`) continues to install the stable **1.1.x** — unaffected users stay unaffected.
- `npx donnyclaude@rc` explicitly opts into **1.2.0-rc.1** — the pruned skill set.

Only explicit opt-in exposes the pruned distribution; the `latest` tag does not move to this pre-release.

## Cooling-off gate

v1.2.0 stable will not ship until a **7-day (7×24h) cooling-off gate** resolves PASSED. The gate window starts at the `npm publish --tag rc` moment and runs exactly 168 hours. During the window three concrete obligations are executed (D-21):

- **(a)** Day 4-5 re-read of `docs/PRUNE-LOG.md` from scratch, pass/fail per row
- **(b)** Fresh-machine `npx donnyclaude@rc` install test end-to-end
- **(c)** Real-workflow use of one borderline-protected skill

Any `prune-regression`-labeled GitHub issue filed in the window, or any failure on (a)/(b)/(c), holds the gate closed. Only after all obligations PASS and the calendar window elapses does v1.2.0 get promoted via `npm dist-tag add donnyclaude@1.2.0 latest`.

## Restoration
